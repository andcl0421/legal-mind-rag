# 테스트/검증 리포트

## 1. 검증 목표
- 핵심 사용자 시나리오가 끊김 없이 동작하는지 확인
- 기능 확장 중 기존 기능 회귀(regression) 없는지 확인

## 2. 백엔드 검증
- 방식: Python `unittest` 실행
- 주요 검증 대상
  - 인증/회원 관련 API
  - 채팅 생성/조회/삭제
  - 알림 생성/조회/읽음
  - 증거파일 업로드/조회/다운로드/삭제
  - 체크리스트 저장/조회
- 결과: 주요 테스트 케이스 통과 확인

## 3. 프론트엔드 검증
- 방식: `vite build`
- 확인 항목
  - 컴파일 에러/타입 에러/번들링 실패 여부
  - 페이지별 렌더링 구조 유지
- 결과: 빌드 성공 확인

## 4. 수동 시나리오 QA
- 홈
  - 감정 선택 시 문구 변경 확인
- 상담실
  - 질문 전송/응답 표시
  - 첨부 패널 검색/카테고리 필터/선택/해제
- 상담내역
  - 탭/검색/세션 선택
  - 체크리스트 체크 저장 및 재진입 복원
  - 세션별 알림 누적 표시
- 증거보관함
  - 업로드/필터/세션연결/다운로드/삭제

## 5. 잔여 리스크
- 운영 환경(배포/모니터링) 검증은 별도 필요
- 대용량 동시접속/부하 테스트는 미포함
- LangGraph 오케스트레이션은 현재 범위 외
