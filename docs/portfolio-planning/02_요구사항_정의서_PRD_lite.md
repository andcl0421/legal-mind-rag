# 요구사항 정의서 (PRD-lite)

## 1. 필수 기능 요구사항
- 홈
  - 오늘의 노동체크 선택 및 상태별 문구 표시
  - 상담실 진입 CTA
- AI 상담실
  - 채팅 송수신
  - 근거 법령 칩/면책 문구 표시
  - 증거 파일 첨부(검색/카테고리 필터/선택 해제)
- 상담내역
  - 카테고리 탭/검색/목록 테이블
  - 상담 선택 시 체크리스트/알림 누적 표시
  - 체크리스트 체크 상태 저장/복원
- 증거보관함
  - 업로드/목록/다운로드/삭제
  - 파일 카테고리/세션 연결
  - 파일 타입별 표시(이미지/PDF/기타)
- 알림
  - 수동 생성/읽음 처리
  - 상담 자동 알림 누적
  - 세션 단위 필터 조회

## 2. 비기능 요구사항
- 반응형 UI(모바일/데스크톱)
- 인증 기반 접근(JWT)
- 기본 안정성: 빌드/테스트 통과 가능한 상태
- 사용자 데이터 분리(본인 세션/파일/알림/체크리스트)

## 3. 범위 정의
### In Scope
- 프론트/백엔드 연동형 상담 서비스
- 상담 후 실행관리(체크리스트/알림)
- 증거 파일 보관 및 상담 연결

### Out of Scope
- 관리자 콘솔
- 멀티 조직 권한 관리
- 대규모 운영 배포/관제
- LangGraph 오케스트레이션

## 4. 성공 기준
- 상담 생성 → 이력 저장 → 체크리스트 표시/저장/복원까지 한 흐름 정상 동작
- 증거 첨부 후 상담 반영 및 보관함 조회 가능
- 세션 선택 시 해당 알림 누적 내역 확인 가능
