# 📊 03. 데이터 및 RAG 상세 설계 (Data & RAG Spec)

## 1. RAG 지식베이스 (Knowledge Base)
AI가 답변의 근거로 삼을 핵심 문서 목록입니다.

### ① 수집 대상 문서 (Internal Data)
- **근로기준법/시행령**: 모든 노무 상담의 기본이 되는 법령 (PDF).
- **남녀고용평등법**: 임신, 출산, 육아휴직 관련 핵심 법령 (PDF).
- **2024~2026 고용노동부 가이드라인**: 실무 지침서 (임금체불 예방, 직장 내 괴롭힘 등).
- **주요 판례집**: 대법원 및 노동위원회 핵심 판결 요약본.

### ② 데이터 전처리 (Chunking Strategy)
- **분할 단위**: 약 500~1,000자 내외 (의미가 끊기지 않도록 문단 단위 분할).
- **중첩(Overlap)**: 이전 문단의 끝부분 100자 정도를 다음 문단에 포함하여 맥락 유지.
- **메타데이터**: 각 조각에 `파일명`, `페이지 번호`, `조항 번호`를 태깅하여 답변 시 출처 표기.

## 2. 데이터베이스 설계 (DB Schema)

### ① User Table (사용자 정보)
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `user_id` | String (PK) | 고유 식별자 (이메일 등) |
| `nickname` | String | 서비스 내 별명 |
| `industry` | String | 업종 (IT, 제조, 서비스 등) |
| `emp_count` | Integer | 상시 근로자 수 (5인 미만 체크용) |
| `join_date` | Date | 입사일 (연차/퇴직금 계산 기준) |
| `region` | String | 거주 지역 (시/군/구 단위) |

### ② Chat History Table (대화 이력)
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `chat_id` | String (PK) | 대화 고유 ID |
| `user_id` | String (FK) | 사용자 연결 키 |
| `question` | Text | 사용자 질문 내용 |
| `answer` | Text | AI 답변 내용 |
| `sources` | JSON | 답변에 참고된 PDF 정보 (파일명, 페이지) |
| `created_at` | DateTime | 대화 생성 시간 |

## 3. 벡터 DB (Vector Store) 구성
- **엔진**: ChromaDB
- **임베딩 모델**: OpenAI `text-embedding-3-small` (또는 최신 모델)
- **검색 방식**: 유사도 기반 검색 (Similarity Search) + MMR (다양성 확보)
