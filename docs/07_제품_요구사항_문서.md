# 07. 제품 요구사항 문서 (PRD)

## 1. 문서 목적
이 문서는 Legal-Mind-RAG의 MVP를 구현하기 위한 제품 요구사항을 정의한다.

이 문서의 목적은 아래 세 가지다.
- 무엇을 만들어야 하는지 명확히 한다.
- 무엇을 아직 만들지 않을지 분명히 한다.
- 기능 구현과 검증의 기준을 제공한다.

---

## 2. 제품 개요

### 2.1 제품명
- **Legal-Mind-RAG**

### 2.2 제품 한 줄 설명
- 일반 근로자가 노동 문제를 질문하면, 법령·가이드라인·판례를 근거로 설명형 답변을 제공하는 AI 노무 상담 도구

### 2.3 해결하려는 문제
- 일반 근로자는 자신의 상황에 어떤 노동법이 적용되는지 알기 어렵다.
- 검색 결과만으로는 최신성, 정확성, 내 상황과의 관련성을 판단하기 어렵다.
- 임금, 휴가, 해고, 임신·출산·육아 관련 노동 이슈는 즉시성 있는 설명이 필요하다.

### 2.4 제품 목표
- 사용자가 자신의 질문에 대해 신뢰 가능한 근거 기반 답변을 받을 수 있게 한다.
- 답변에 출처와 면책 문구를 포함해 오해와 환각을 줄인다.
- 상담 내용을 세션 단위로 저장해 이어서 확인할 수 있게 한다.

---

## 3. 대상 사용자

### 3.1 핵심 사용자
- 노동법 비전공자인 일반 근로자
- 자신의 상황에 맞는 법 적용 여부를 빠르게 알고 싶은 사용자
- 노무사 상담 전 사전 판단과 정리가 필요한 사용자

### 3.2 대표 사용 시나리오
- "육아휴직 급여는 얼마까지 받을 수 있나요?"
- "저는 5인 미만 사업장인데 연차가 있나요?"
- "이 상황이 부당해고에 해당할 수 있나요?"
- "주휴수당을 받을 수 있는 조건이 뭔가요?"

---

## 4. MVP 범위
MVP는 아래 경험 하나에 집중한다.

1. 사용자가 질문을 입력한다.
2. 시스템이 관련 법령/가이드라인/판례를 검색한다.
3. AI가 근거 기반 답변을 생성한다.
4. 답변에 출처와 면책 문구를 함께 표시한다.
5. 상담 내역을 저장한다.

### 4.1 MVP 포함 항목
- 채팅 기반 상담 인터페이스
- 질문 입력 및 응답 표시
- 근거 문서 검색 기반 답변 생성
- 출처 표기
- 면책 문구 표시
- 상담 세션 저장
- 메시지 저장
- 답변 메타 저장

### 4.2 MVP 제외 항목
- 맞춤형 알림 발송
- 소셜 로그인
- 전문가 연계
- 증거 보관함
- 완성형 PDF 리포트
- 고도화된 사용자 프로필 진단

---

## 5. 기능 요구사항

### FR-01. 질문 입력
- 사용자는 상담 화면에서 질문을 입력할 수 있어야 한다.
- 질문은 하나의 세션 안에 메시지로 저장되어야 한다.
- 빈 입력은 전송할 수 없어야 한다.

**수용 기준**
- 사용자가 질문을 보내면 새 메시지로 표시된다.
- 저장된 질문은 동일 세션에서 다시 조회할 수 있다.

### FR-02. 상담 세션 관리
- 시스템은 상담 세션을 생성하고 식별할 수 있어야 한다.
- 하나의 세션에는 여러 메시지가 순서대로 저장되어야 한다.

**수용 기준**
- 새 상담 시작 시 새 세션이 생성된다.
- 각 메시지는 세션 내 순서를 가진다.

### FR-03. 근거 문서 검색
- 시스템은 질문과 관련된 문서 조각을 검색할 수 있어야 한다.
- 검색 대상은 법령, 가이드라인, 판례 자료여야 한다.

**수용 기준**
- 응답 생성 전에 관련 문서 검색 단계가 수행된다.
- 검색 결과는 답변 생성 과정에서 참조된다.

### FR-04. AI 답변 생성
- 시스템은 사용자 질문과 검색된 근거를 바탕으로 설명형 답변을 생성해야 한다.
- 답변은 비전공자도 이해 가능한 표현을 우선해야 한다.

**수용 기준**
- 답변이 단순 일반론이 아니라 질문 맥락에 반응해야 한다.
- 답변은 문장 형태로 완결되어야 한다.

### FR-05. 출처 표기
- 시스템은 답변에 근거 출처를 포함해야 한다.
- 출처는 가능한 경우 문서명, 페이지, 조항 등의 형태로 표시되어야 한다.

**수용 기준**
- 사용자 응답 화면에서 출처 정보가 확인 가능하다.
- 출처가 없는 답변은 정상 응답으로 간주하지 않는다.

### FR-06. 면책 문구 표시
- 시스템은 모든 상담 응답에 법적 효력 부재와 전문가 상담 권고 성격의 면책 문구를 포함해야 한다.

**수용 기준**
- 모든 AI 응답에 면책 문구가 함께 저장되거나 표시된다.

### FR-07. 상담 이력 저장
- 시스템은 세션, 사용자 메시지, AI 응답, 응답 메타를 저장해야 한다.

**수용 기준**
- 같은 세션을 다시 조회했을 때 과거 대화가 유지된다.
- 답변별 메타 정보가 연결되어 저장된다.

### FR-08. 최소 사용자 맥락 반영
- 시스템은 필요한 경우 질문 해석에 사용자 조건을 반영할 수 있어야 한다.
- 반영 가능한 예시는 근로자 수 규모, 업종, 재직 상태다.

**수용 기준**
- 사용자 조건이 제공된 경우 답변 내용에 해당 조건이 반영될 수 있다.

---

## 6. 비기능 요구사항

### NFR-01. 신뢰성
- 답변은 근거 기반이어야 하며, 출처 없는 단정적 응답을 지양해야 한다.

### NFR-02. 이해 가능성
- 답변은 비전공자도 이해할 수 있는 표현을 사용해야 한다.
- 지나치게 법률 문구만 나열해서는 안 된다.

### NFR-03. 개인정보 보호
- 실명은 필수 수집 항목이 아니어야 한다.
- 상담에 불필요한 민감정보는 최소화해야 한다.
- 비밀번호 등 주요 정보는 안전한 방식으로 저장해야 한다.

### NFR-04. 확장 가능성
- 현재는 작은 MVP 구조로 시작하되, 이후 사용자 프로필/알림/리포트/문서 계층 구조로 확장 가능해야 한다.

### NFR-05. 추적 가능성
- 답변과 그에 연결된 메타 정보는 추후 검토 가능하도록 저장되어야 한다.

---

## 7. 사용자 흐름 요구사항

### 7.1 기본 상담 흐름
1. 사용자가 상담 화면에 진입한다.
2. 질문을 입력한다.
3. 시스템이 질문을 저장한다.
4. 시스템이 관련 문서를 검색한다.
5. AI가 답변을 생성한다.
6. 답변과 출처와 면책 문구를 표시한다.
7. 결과를 세션에 저장한다.

### 7.2 재조회 흐름
1. 사용자가 기존 상담 세션에 다시 진입한다.
2. 이전 메시지와 응답을 확인할 수 있다.

## 8. 데이터 요구사항

### 8.1 MVP에서 필요한 핵심 데이터
- `chat_sessions`
- `messages`
- `answer_metas`

### 8.2 장기적으로 필요한 데이터
- `users`
- `user_profiles`
- `companies`
- `user_lifecycles`
- `documents`
- `doc_hierarchies`
- `document_chunks`
- `answer_traces`
- `auth_providers`
- `user_notifications`

### 8.3 문서 데이터 요구사항
- 법령/가이드라인/판례 문서는 검색 가능한 단위로 분할되어야 한다.
- 각 문서 조각에는 출처 표기를 위한 메타데이터가 있어야 한다.

---

## 9. 외부 의존성 요구사항
- LLM 기반 응답 생성을 위한 모델 연동이 필요하다.
- 법령/가이드라인/판례 데이터 수집 경로가 필요하다.
- 장기적으로는 벡터 검색 저장소 구성이 필요하다.

---

## 10. 성공 기준
MVP가 성공했다고 보기 위한 최소 조건은 아래와 같다.

1. 사용자가 질문을 입력하고 답변을 받을 수 있다.
2. 답변에 근거 출처가 표시된다.
3. 답변에 면책 문구가 포함된다.
4. 상담 내역이 저장되고 다시 조회된다.

---

## 11. 향후 확장 요구사항
아래는 본 문서 기준의 후속 단계 요구사항이다.

- 사용자 프로필 기반 법 적용 자동화
- 맞춤형 노무 이슈 알림
- PDF 리포트 생성
- 문서 원문 뷰어와 하이라이트
- 전문가 전달용 브리핑 기능
- 소셜 로그인

---

## 12. 한 문장 요약
Legal-Mind-RAG MVP의 요구사항은 "근로자의 질문에 대해 근거를 제시하는 AI 상담 응답을 생성하고, 그 대화를 저장해 다시 확인할 수 있게 하는 것"이다.
