# 노무톡톡 트러블슈팅 요약 (Backend / RAG / LLM / Frontend)

이 문서는 실제 구현 과정에서 발생했던 주요 이슈와, 원인/수정 방식/수정 이유를 정리한 기록입니다.

---

## 1) Backend

## 1-1. UUID 타입 충돌로 상담 조회 실패
- 증상  
  - 채팅 전송/조회 테스트 중 SQLAlchemy 오류 발생  
  - 에러: `AttributeError: 'str' object has no attribute 'hex'`
- 원인  
  - `chat_session_id`(UUID 컬럼) 비교 시 문자열을 그대로 조건에 넣어 타입 불일치 발생
- 수정  
  - 문자열 `chat_session_id`를 먼저 UUID로 파싱 후 쿼리 비교하도록 변경
  - 위치: `app/services/chat_service.py`
- 이유  
  - DB 컬럼 타입과 동일한 타입으로 비교해야 ORM 바인딩/쿼리 안정성이 확보됨

---

## 1-2. 기존 DB 스키마와 신규 기능 컬럼 불일치
- 증상  
  - 새 기능(증거 카테고리/세션 연결) 적용 후 기존 DB에서 컬럼 부재 가능성
- 원인  
  - 개발 중 기능 확장으로 `user_files` 등에 신규 컬럼 추가 필요
- 수정  
  - 앱 시작 시 컬럼 존재 여부 검사 후 `ALTER TABLE`로 보정 로직 추가
  - 위치: `app/main.py`
- 이유  
  - 로컬/기존 DB에서도 앱이 바로 기동되도록 마이그레이션 격차를 완화하기 위함

---

## 1-3. 상담별 알림 조회 필요 (기획 반영)
- 증상  
  - 상담별 체크리스트/알림 패널 구현 시, 해당 세션 알림만 필터링이 필요
- 원인  
  - 기존 `alerts` API는 `source`/`unread_only` 위주였고 세션 필터가 없음
- 수정  
  - `GET /api/v1/alerts`에 `chat_session_id` 필터 파라미터 추가
  - 위치: `app/api/v1/endpoints/alerts.py`
- 이유  
  - “상담 1건 선택 → 해당 상담의 알림 누적” UX를 정확히 구현하기 위해 필요

---

## 1-4. 체크리스트 상태 저장 기능 부재
- 증상  
  - “먼저 할 일/준비할 서류” 체크박스 상태가 새로고침 시 사라짐
- 원인  
  - 클라이언트 임시 상태만 있었고 서버 저장 구조가 없음
- 수정  
  - `user_checklist_items` 모델 및 `checklist` API 신규 추가
  - `POST /api/v1/checklist` (upsert), `GET /api/v1/checklist` 구현
  - 위치: `app/models/user_checklist.py`, `app/api/v1/endpoints/checklist.py` 등
- 이유  
  - 상담별 진행 상태를 누적/복원해 실사용 가능한 업무 흐름을 만들기 위함

---

## 2) RAG

## 2-1. 후속 질문 맥락 약화로 답변 주제 이탈
- 증상  
  - “그럼 이제 뭐부터 해야 돼?” 같은 후속 질문에서 앞선 상담 맥락 이탈 가능
- 원인  
  - 짧은 후속 질문은 단독 질의로 검색 시 정보 밀도가 낮아 관련 청크가 흔들릴 수 있음
- 수정  
  - 후속질문 판단 후, 세션 카테고리/최근 사용자 메시지를 포함해 검색 쿼리 보강
  - 위치: `app/services/chat_service.py` (`_build_search_query`)
- 이유  
  - 대화형 상담에서는 단일 턴보다 “연속 맥락” 기반 검색이 정확도에 유리

---

## 2-2. 첨부 증거가 검색 맥락에 반영되지 않음
- 증상  
  - 사용자가 파일을 첨부했는데 답변에 증거 맥락이 약하거나 미반영
- 원인  
  - 기존 검색/답변 파이프라인은 질문 본문 중심, 첨부 메타정보 전달 경로 부재
- 수정  
  - 채팅 요청에 `evidence_file_ids` 추가
  - 서버에서 사용자 소유 파일 메타를 조회해 `effective_content`로 검색 컨텍스트 보강
  - 답변 본문에도 “검토한 첨부 증거”를 명시
- 이유  
  - 상담 품질과 사용자 신뢰를 위해 “첨부했으면 반영된다”는 피드백 루프가 중요

---

## 3) LLM

## 3-1. 출력 구조 편차(형식 흔들림) 대응
- 증상  
  - 모델 응답이 항상 같은 JSON 구조로 오지 않을 수 있음
- 원인  
  - 생성형 모델 특성상 포맷 편차/파싱 실패 케이스 존재
- 수정  
  - JSON 추출/정제 로직(펜스 블록/부분 JSON 대응)과 fallback 답변 경로 구성
  - 섹션 헤딩 허용 목록, citation 유효성 검증 적용
- 이유  
  - 서비스 안정성을 위해 “모델 응답 실패 시에도 답변 생성” 가능한 방어 로직 필요

---

## 3-2. 모델 의존성만으로는 실행 항목 전달 부족
- 증상  
  - 사용자 기대는 “답변”뿐 아니라 “다음 행동/필요 서류”까지 포함
- 원인  
  - LLM 자유 응답만으로는 액션아이템 일관성이 떨어질 수 있음
- 수정  
  - 규칙 기반 `_extract_action_items`를 병행해 `next_actions`, `required_docs`, `deadlines` 생성
  - 자동 알림(`summary`, `document`)으로 후속 관리 가능하게 연결
- 이유  
  - 상담 서비스의 실효성을 위해 “실행 가능한 출력”을 구조화해 제공

---

## 4) Frontend

## 4-1. 페이지 텍스트 깨짐(인코딩 문제)
- 증상  
  - 일부 파일에서 한글이 깨져 `???` 또는 mojibake 형태로 노출
- 원인  
  - 편집/저장 과정에서 인코딩 혼선이 발생
- 수정  
  - 해당 페이지를 안전하게 재작성(ASCII-safe + 정상 한글 문자열로 교체)
  - 주요 대상: `Home.jsx`, `History.jsx`, `Chat.jsx`, `MyPage.jsx` 관련 구간
- 이유  
  - UI 신뢰성과 유지보수성을 위해 문자열 깨짐을 근본 정리해야 함

---

## 4-2. 첨부 UX가 파일 수 증가 시 비효율
- 증상  
  - 첨부 파일이 많아지면 선택이 어려움
- 원인  
  - 단순 리스트 체크박스만 제공
- 수정  
  - 첨부 패널에 파일명 검색 + 카테고리 quick filter 추가
  - 선택 파일 칩 표시/해제로 가시성 향상
- 이유  
  - 실사용 시 탐색 비용을 줄여야 첨부 기능 활용률이 올라감

---

## 4-3. 전체 페이지 스크롤 vs 채팅창 내부 스크롤
- 증상  
  - 답변 길어질 때 페이지 전체가 내려가 사용성 저하
- 원인  
  - 레이아웃 높이/overflow 설정이 대화창 중심으로 고정되지 않음
- 수정  
  - 채팅 레이아웃을 `min-h-0`, 내부 `overflow-y-auto` 구조로 조정
- 이유  
  - 메신저형 UI는 “대화 영역만 스크롤”이 사용자 기대와 일치

---

## 4-4. 상담내역에서 실행 관리(체크리스트) 연결 부족
- 증상  
  - 상담 목록은 보이나, 상담별 실행 항목 확인/관리 흐름이 약함
- 원인  
  - 목록과 자동 알림 데이터가 분리되어 있었음
- 수정  
  - 상담 row 선택 시 우측 패널에서:
    - 먼저 할 일
    - 준비할 서류
    - 알림 누적
    - 체크박스 저장 상태
    를 통합 표시
- 이유  
  - “상담 → 실행”으로 이어지는 제품 목적에 맞는 핵심 UX 보완

---

## 5) 개발 환경/검증 이슈

## 5-1. 로컬 명령 실행 정책 문제
- 증상  
  - `npm` PowerShell 스크립트 실행 정책 에러
- 수정  
  - `npm.cmd run build`로 우회 실행

## 5-2. Python 실행 경로 차이
- 증상  
  - `python`/`py` 경로 인식 이슈
- 수정  
  - `.venv\\Scripts\\python.exe` 직접 지정해 테스트 실행

## 5-3. 회귀 검증
- 수행  
  - 백엔드: `unittest` 스위트 반복 실행
  - 프론트: `vite build` 반복 검증
- 목적  
  - 기능 추가 중 기존 상담/인증/알림 흐름 깨짐 방지

---
