# 08. 기능 요구 정의서

## 1. 문서 목적
이 문서는 Legal-Mind-RAG MVP에서 구현해야 할 기능 요구를 기능 단위로 정의한다.

이 문서는 아래 질문에 답하기 위해 사용한다.
- 지금 어떤 기능을 만들어야 하는가?
- 각 기능은 어떤 입력과 결과를 가져야 하는가?
- 구현 완료를 어떤 기준으로 판단할 것인가?

---

## 2. 적용 범위
본 문서는 현재 MVP 범위에 적용된다.

현재 MVP의 중심 기능은 아래 한 흐름이다.
1. 질문 입력
2. 근거 문서 검색
3. AI 답변 생성
4. 출처 및 면책 문구 표시
5. 상담 이력 저장

---

## 4. 상세 기능 요구

### FR-01. 질문 입력
**목적**
- 사용자가 상담을 시작할 수 있어야 한다.

**설명**
- 사용자는 텍스트 형태의 질문을 입력할 수 있어야 한다.
- 질문은 전송 즉시 대화 흐름에 반영되어야 한다.

**입력**
- 사용자 질문 텍스트

**출력**
- 세션 내 사용자 메시지 생성

**조건**
- 빈 문자열은 전송할 수 없다.
- 정상 전송 시 메시지 저장 단계로 이어진다.

**수용 기준**
- 질문 전송 후 채팅창에 사용자 메시지가 나타난다.
- 동일 세션을 다시 열었을 때 해당 질문이 유지된다.

### FR-02. 상담 세션 생성 및 관리
**목적**
- 하나의 상담 흐름을 세션 단위로 구분하고 저장하기 위함이다.

**설명**
- 시스템은 새 상담 시작 시 새로운 세션을 생성할 수 있어야 한다.
- 각 메시지는 특정 세션에 속해야 한다.

**입력**
- 사용자 진입 또는 새 상담 시작 요청

**출력**
- `chat_session_id`

**조건**
- 세션에는 여러 메시지가 순서대로 연결되어야 한다.

**수용 기준**
- 새 상담을 시작하면 새로운 세션 식별자가 생성된다.
- 각 메시지는 세션 내 순서를 가진다.

### FR-03. 관련 문서 검색
**목적**
- 답변이 일반 상식이 아니라 근거 자료에 기반하도록 하기 위함이다.

**설명**
- 시스템은 질문과 관련된 문서 조각을 검색해야 한다.
- 검색 대상은 법령, 가이드라인, 판례 자료여야 한다.

**입력**
- 사용자 질문

**출력**
- 관련 문서 조각 목록

**조건**
- 검색 결과는 답변 생성 단계에서 참조 가능해야 한다.

**수용 기준**
- 답변 생성 전에 검색 단계가 호출된다.
- 검색 결과가 없는 경우에도 시스템은 실패를 처리하거나 제한된 응답 정책을 적용할 수 있어야 한다.

### FR-04. AI 답변 생성
**목적**
- 사용자 질문에 대해 이해 가능한 설명형 답변을 제공하기 위함이다.

**설명**
- 시스템은 질문과 검색된 근거를 함께 사용해 답변을 생성해야 한다.
- 답변은 비전공자도 이해 가능한 표현을 우선해야 한다.

**입력**
- 사용자 질문
- 관련 문서 조각
- 선택적 사용자 맥락

**출력**
- AI 답변 텍스트

**조건**
- 답변은 질문과 무관한 일반 설명에 머물러서는 안 된다.

**수용 기준**
- 답변은 완결된 문장으로 생성된다.
- 질문 주제와 연결된 설명을 포함한다.

### FR-05. 출처 표시
**목적**
- 사용자가 답변의 근거를 확인할 수 있게 하기 위함이다.

**설명**
- 시스템은 답변과 함께 출처 정보를 제공해야 한다.
- 출처는 문서명, 페이지, 조항 번호 등 가능한 범위의 메타데이터를 포함한다.

**입력**
- 검색 결과 메타데이터

**출력**
- 사용자에게 표시되는 출처 정보

**수용 기준**
- 사용자 응답 화면에서 출처가 확인 가능하다.
- 출처 없는 응답은 정상 완료 상태로 보지 않는다.

### FR-06. 면책 문구 표시
**목적**
- AI 답변이 법적 자문으로 오인되지 않도록 하기 위함이다.

**설명**
- 모든 AI 답변에는 면책 문구가 포함되어야 한다.

**입력**
- AI 응답 생성 결과

**출력**
- 답변과 함께 표시되거나 저장되는 면책 문구

**수용 기준**
- 모든 상담 응답에 면책 문구가 확인된다.

### FR-07. 상담 이력 저장
**목적**
- 사용자가 과거 상담을 다시 볼 수 있게 하고, 시스템이 대화를 추적할 수 있게 하기 위함이다.

**설명**
- 시스템은 세션, 사용자 메시지, AI 메시지, 답변 메타를 저장해야 한다.

**입력**
- 세션 정보
- 메시지 정보
- 답변 메타 정보

**출력**
- 저장된 상담 기록

**수용 기준**
- 세션 재조회 시 대화 내용이 유지된다.
- 메시지와 답변 메타가 연결되어 저장된다.

### FR-08. 상담 이력 조회
**목적**
- 기존 상담 내용을 다시 확인하기 위함이다.

**설명**
- 사용자는 저장된 세션에 다시 진입해 과거 메시지를 볼 수 있어야 한다.

**입력**
- 세션 식별자

**출력**
- 세션 내 메시지 목록

**수용 기준**
- 저장된 세션을 다시 열면 기존 메시지가 보인다.

### FR-09. 최소 사용자 맥락 반영
**목적**
- 동일 질문이라도 사용자 조건에 따라 답변이 달라질 수 있기 때문이다.

**설명**
- 시스템은 필요한 경우 근로자 수 규모, 업종, 재직 상태 같은 조건을 반영할 수 있어야 한다.

**입력**
- 사용자 조건 정보

**출력**
- 맥락 반영된 답변

**수용 기준**
- 조건이 제공될 경우 답변 내용에 해당 조건이 반영될 수 있다.

## 5. 기능 간 흐름 관계

1. `FR-01 질문 입력`
2. `FR-02 세션 관리`
3. `FR-03 관련 문서 검색`
4. `FR-04 AI 답변 생성`
5. `FR-05 출처 표시`
6. `FR-06 면책 문구 표시`
7. `FR-07 상담 이력 저장`
8. `FR-08 상담 이력 조회`

`FR-09 최소 사용자 맥락 반영`은 검색과 답변 생성 단계에서 선택적으로 개입한다.

---

## 6. MVP 제외 기능
아래 기능은 현재 기능 요구 범위에 포함하지 않는다.

- 맞춤형 알림
- 소셜 로그인
- 전문가 연계
- 증거 보관함
- 완성형 PDF 리포트
- 고도화된 라이프사이클 진단

---

## 7. 한 문장 요약
현재 기능 요구의 핵심은 "근거 기반 질문-응답 상담 흐름을 만들고, 그 결과를 다시 확인할 수 있게 저장하는 것"이다.
