# 📄 01. 프로젝트 상세 기획 및 요구사항 (Detailed Spec)

## 1. 서비스 정의 및 가치
- **서비스명**: Legal-Mind-RAG
- **핵심 목표**: 비전공자 및 일반 근로자가 복잡한 노무/세무 지식을 AI를 통해 쉽고 정확하게 확인하는 '지능형 비서'.
- **차별점**: 단순 챗봇이 아닌, 최신 법령 PDF와 공공데이터 API를 결합한 '근거 기반' 상담.

## 2. 핵심 기능 (Features)
### ① AI 노무/세무 컨설팅 (RAG)
- 사용자의 질문(예: "육아휴직 급여 얼마인가요?")에 대해 최신 가이드북을 찾아 답변.
- 답변 하단에 근거가 된 법 조항 번호와 페이지를 반드시 표기.
- "법적 효력이 없음"을 알리는 면책 공고(Disclaimer) 자동 부착.

### ② 맞춤형 노무 이슈 알림
- 최신 개정 법령이나 고용노동부 공지사항을 요약하여 알림 제공.
- 예: "2026년부터 육아휴직 급여가 월 최대 250만 원으로 인상됩니다!"

### ③ 사용자 맞춤 진단 (회원제 기반)
- 로그인한 사용자의 정보를 바탕으로 법 적용 여부 자동 판단.
- 예: "5인 미만 사업장이시므로 일부 근로기준법 조항이 다르게 적용됩니다."

## 3. 정보 수집 및 개인정보 보호 (Data Privacy & Management)

### ① 수집 항목 및 목적
- **기본 정보**: 닉네임, 이메일(ID), 비밀번호 (서비스 이용 및 본인 확인용)
- **상담 맞춤 정보**: 업종, 상시 근로자 수(5인 미만/이상), 입사일, 지역(시/군/구)
  - *목적: 사용자의 상황에 맞는 정확한 법적 적용 범위 판단 및 맞춤 답변 제공*

### ② 로그인 및 인증 방식
- **자체 회원가입**: 이메일 기반 인증 및 보안 비밀번호 저장.
- **간편 로그인**: 카카오/구글 API 연동을 통한 접근성 향상 (추후 확장).

### ③ 개인정보 보호 4대 원칙 (Privacy Policy)
1. **실명 비수집 (Anonymity)**: 이름 대신 닉네임을 사용하여 개인 식별 최소화.
2. **정보 최소화 (Minimization)**: 상담에 불필요한 상세 주소(번지수 등)는 수집하지 않음.
3. **민감 정보 필터링 (PII Masking)**: AI 모델로 데이터를 전송하기 전, 채팅창에 입력된 개인정보(전화번호 등) 자동 마스킹 처리.
4. **데이터 암호화 (Encryption)**: 비밀번호 및 주요 정보는 해시(Hash) 암호화하여 DB 저장.

## 4. 데이터 및 외부 API 연동 계획
- **법령 데이터**: 국가법령정보센터(law.go.kr) PDF 수집.
- **계산 로직**: 근로복지공단/고용노동부 공공데이터 API 연동(퇴직금/실업급여 요율 등).
- **AI 엔진**: OpenAI GPT-4o + LangChain.

## 5. 화면 구성 (UI/UX Plan)
### [1] 메인 페이지 (Dashboard)
- 요약된 내 근로 정보 카드.
- 실시간 주요 노무 이슈 뉴스 피드.
- 빠른 상담 시작 버튼.

### [2] 상담 페이지 (Chat Room)
- AI와의 채팅 인터페이스.
- 우측 사이드바: 상담의 근거가 된 '참조 문서' 뷰어.
- 하단: 간이 계산기 위젯(연차/주휴수당용).

### [3] 마이페이지/리포트
- 과거 상담 이력 관리 및 PDF 저장 기능.
- 내 정보(업종, 근로자 수 등) 수정.
